# 参与贡献

感谢你愿意改进 OPC Toolkit。

## 提交建议

1. 先搜索现有 Issue，避免重复。
2. 描述真实使用场景、当前问题和期望结果。
3. 新技能应解决反复出现的独立任务，不要只为了分类整齐而增加。
4. 不要提交客户资料、未授权案例、个人信息、Token、密钥或付费报告原文。

## 本地验证

```bash
python3 scripts/validate_bundle.py
bash scripts/smoke_test.sh
```

提交 Pull Request 时，请说明：

- 改了什么；
- 为什么需要；
- 如何验证；
- 是否影响已有技能的触发或输出。

## 内容原则

- 区分事实、推断、假设和建议。
- 时效性信息应要求运行时核验，不要固化成长期事实。
- 示例使用虚构品牌或通用占位符。
- 引用第三方内容时遵守其许可证和引用要求。
