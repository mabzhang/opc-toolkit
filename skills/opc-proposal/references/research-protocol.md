# OPC Research Protocol

Use this protocol before making claims about current markets, platforms, competitors, KOLs, budgets, prices, regulations, or campaign performance.

## Source Priority

Use this order unless the user says otherwise:

1. User-uploaded or user-specified local files.
2. User-approved web research for topics not covered by local files.
3. Public web data only when current information is required and the user has allowed it.

If local files cover a topic, do not replace or override them with web data unless the user explicitly asks for external validation. When local files do not cover a required topic, ask whether web search is allowed or mark the conclusion as an assumption.

## Source Modes

Confirm one mode for every major analysis block before research:

| Mode | Meaning | Use When |
|---|---|---|
| 仅本地上传 | Use only uploaded/user-specified files | The user has provided category, product, meeting or prior proposal data and does not authorize web replacement |
| 仅网络公开 | Use public web sources only | The user has no local data for this block and authorizes web research |
| 本地为主+网络补充 | Local data is the base; web only fills missing or current details | The user provides core data but asks for trend/platform/competitor/KOL updates |
| 网络为主+本地校验 | Web research drives the block; local files validate brand/product facts | The user's local files are not sufficient for market facts but still constrain product/brand claims |

If the mode is not confirmed, do not proceed with detailed research. Ask or state a temporary assumption in the confirmation note.

## Source Tiers

| Tier | Use For | Examples | Confidence |
|---|---|---|---|
| A | Official facts and rules | Brand official site, platform ad docs, regulator notices, financial filings | High |
| B | Industry and platform signals | QuestMobile, iResearch, platform white papers, industry associations, verified media | Medium-high |
| C | Market behavior and content examples | Xiaohongshu, Douyin, Weibo, ecommerce reviews, creator posts, ad libraries | Medium |
| D | Directional clues | Agency blogs, unverified rankings, anecdotal comments | Low |

Prefer Tier A/B for factual claims. Use Tier C for consumer language, content patterns and examples. Use Tier D only as weak directional evidence.

## Evidence Labels

Label conclusions with one of:
- **Fact:** verified by current source.
- **Observed Pattern:** repeated across multiple examples.
- **Inference:** reasoned from available evidence, not directly verified.
- **Assumption:** used to keep work moving and must be confirmed.
- **Risk:** possible blocker, compliance issue, or execution uncertainty.

## Research Minimums

For a complete proposal:
- Follow the confirmed source mode for each block.
- Industry/category: use local files when provided; use recent web sources only when authorized or required for uncovered current context.
- Competitors: 3-5 competitors for landscape scans, or 2-3 for deep campaign analysis, with source or assumption labels.
- Consumer voice: use real local/user-provided/social/ecommerce/search examples when available; do not invent quotes.
- Platform specs/rules: verify against official platform or current help-center sources when the output will guide production and the user authorizes/current rules are required.
- Budget benchmarks: state geography, platform, time window and whether figures are market benchmarks, local data, or internal assumptions.

## Citation Format

When citing web research, include:
- Source name
- Link
- Publication or access date
- What the source supports

Use concise citations. Do not paste long excerpts.

## Handling Gaps

When data is unavailable:
- Say what was checked.
- State the closest usable proxy.
- Mark the resulting conclusion as inference or assumption.
- Add the missing item to the information-gap list if it could affect strategy, budget, KPI or schedule.
