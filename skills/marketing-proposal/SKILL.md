# Marketing Proposal — 客户提案写作

把 Brief、研究、策略和执行计划整理为客户可读、可汇报、可决策的正式营销方案。重点是说服逻辑、结构完整、证据清楚和动作可落地。

## 写作原则

1. 先结论后展开：客户先看到我们推荐什么和为什么。
2. 用“背景 -> 洞察 -> 挑战 -> 策略 -> 创意/业务落地 -> 预算/KPI -> 排期”的逻辑链。
3. 每个建议写明目标连接：为了什么、做什么、怎么衡量。
4. 数据、竞品动作、用户原话、平台规则必须有来源；无来源则标注为假设。
5. 预算和 ROI 不凭空承诺；给测算口径、变量和风险。
6. 文风专业、清晰、有判断，避免堆砌形容词。
7. 默认输出 Markdown `.md` 文档。若用户要求 PPT 或其它格式，把它当作额外交付物。
8. 章节顺序按用户本次业务优先级动态调整，不把历史案例或固定 16 模块当作最终目录。

## 提案层级

| 类型 | 适用场景 | 推荐结构 |
|---|---|---|
| 精简方案 | 方向确认、内部沟通 | 6-8 个章节，突出判断和下一步 |
| 完整方案 | 客户提案、比稿 | 12-16 个章节，含研究、策略、执行、预算 |
| 执行方案 | 已定策略后的落地 | 重点写任务、排期、验收和风险 |
| 复盘/优化 | 投放后或运营中 | 数据表现、问题诊断、优化动作 |

## 动态章节模块

以下模块用于组装正式提案，不是固定目录。先看用户这次是直播、品牌 Campaign、商品卡、千川、KOL 媒介、年度运营还是研究分析，再决定前置顺序。

1. 封面信息：品牌、项目、日期、版本。
2. 执行摘要：一句话策略、关键机会、推荐打法。
3. Brief 解构：目标、受众、产品、限制、关键矛盾。
4. 资料来源与假设：本地资料、网络资料、来源矩阵、信息缺口。
5. 市场与竞品：行业趋势、品类格局、竞品启示。
6. 消费者洞察：人群、痛点、购买链路、内容触发。
7. 核心挑战：本次项目必须解决的关键问题。
8. 策略方向：核心主张、差异化、证据链。
9. 创意主题：主题、解读、视觉/文案调性、内容支柱。
10. 重点业务方案：把用户第一优先级写深，例如直播、Campaign、商品卡、千川、KOL/媒介或店铺运营。
11. 支撑业务方案：只写与主目标强相关的承接模块。
12. KOL/媒介：达人组合、平台角色、投放优化。
13. 预算与 KPI：预算结构、目标框架、测量方式。
14. 时间表与任务：里程碑、责任分工、验收标准。
15. 风险预案：关键风险、触发信号、B 方案。
16. 附录：来源、计算、假设、信息缺口。

## 说服力检查

- 不是“市场很大”，而是说明具体增长点和本项目可抓住的部分。
- 不是“年轻人喜欢”，而是说明哪个细分人群、什么场景、什么证据。
- 不是“做内容矩阵”，而是说明每个内容支柱承担什么任务。
- 不是“找达人种草”，而是说明达人层级、垂类、人设和验收标准。
- 不是“提升 ROI”，而是说明预算口径、转化链路和优化变量。

## 规则

- 如果用户要求“4A 全案”，优先使用 `$opc-proposal` 的动态融合框架和 `proposal-blueprint.md` 的策略完整性模块。
- 如果素材不足，仍可出 V0.1 方向稿，但必须列假设和需确认事项。
- 避免在提案正文里写过多过程说明；把来源和假设放附录。
